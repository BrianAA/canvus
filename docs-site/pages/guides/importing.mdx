# HTML/CSS Document Importing

Canvus provides an `importHTMLDocument` function that parses complete HTML documents (or fragments) and converts them into content nodes that can be mounted in the workspace.

## Basic Usage

```typescript
import { Workspace, importHTMLDocument } from 'canvus'

const ws = new Workspace(container, callbacks)

// Import an HTML string
const nodes = importHTMLDocument('<div class="card"><h1>Title</h1><p>Body</p></div>')

// Mount the imported nodes
for (const node of nodes) {
  ws.addNode(node)
}
```

## Options

```typescript
interface ImportHTMLOptions {
  // Base ID prefix for generated node IDs
  idPrefix?: string
  // Whether to flatten nested structures
  flatten?: boolean
}
```

## How It Works

The importer:

1. Parses the HTML string into a DOM tree
2. Walks the element tree to identify content boundaries
3. Generates `WebHTMLNode` objects with unique IDs
4. Preserves parent-child relationships via `parentId` and `childIds`
5. Extracts raw markup for each node

import { Callout } from 'nextra/components'

<Callout type="info">
  The importer does not execute JavaScript in the imported HTML. Scripts are stripped following the same policy described in [ADR-0005](/architecture/adr/0005-stylesheets).
</Callout>

## Stylesheet Injection

To apply stylesheets from the imported document, inject them into the Shadow DOM:

```typescript
const shadowMount = ws.getShadowMount()

// Inject Tailwind CSS or custom styles
shadowMount.injectCSS(`
  .card { padding: 24px; border-radius: 12px; }
  .btn-primary { background: #6366f1; color: white; }
`)
```

Styles are scoped to the Shadow Root and won't affect the host application.
