## 1. Project Overview & Architectural Vision

Canvus is a headless, framework-agnostic, vanilla TypeScript SDK for an interactive layout editor workspace. It runs out-of-the-box in standard web browsers without Electron flags or iframe isolation.

### The Twin-Layer Architecture
To avoid the security, scroll-sync, and styling isolation limitations of nested `<iframe>`s, Canvus splits the workspace into two highly-synchronized layers:

1. **The Projection Mutation Layer (Shadow DOM):**
   Managed by [ShadowMount](file:///Users/balfaro01/Documents/GitHub/canvus/src/shadow-mount.ts). It injects raw user HTML/CSS source inside an isolated browser `ShadowRoot` (`mode: 'open'`). This protects the editor's host shell styles from pollution while leveraging the browser's native layout engine (Blink/WebKit/Gecko) for high-performance text wrapping, Flexbox, CSS Grid, and Tailwind layout parsing.
2. **The Viewport Surface Layer (HTML5 Canvas):**
   Managed by [OverlayRenderer](file:///Users/balfaro01/Documents/GitHub/canvus/src/renderer.ts). A clean canvas overlay coordinates selection highlights, resize anchors, spacing adjusters (paddings/margins), alignment guidelines, layout status badges, and mouse gesture hits.

## 2. Workspace & Folder Structure

The project repository is split into two primary operational workspaces:

### A. The Workspaces Defined
1.  **SDK Package Core Workspace (`src/`)**: 
    - Contains the source code of the headless layout editor engine.
    - Built using pure, vanilla TypeScript with strict types and zero external runtime framework dependencies.
    - Outputs a compilation bundle to `dist/` (via `npm run build`) containing the ESM modules and declaration typings.
2.  **Dev Workbench Workspace (`demo/`)**:
    - Serves as the interactive testing and integration environment for local development.
    - Contains an `index.html` file that directly imports and instantiates the `Workspace` class from the compiled `dist/` workspace.
    - Runs on port `3000` via `npm run demo` for visual validation, feature testing, and AST logging.

### C. Source Code Modules & File Roles

Here is the blueprint of what each TypeScript source module inside `src/` owns:

*   **[types.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/types.ts)**: *Geometric & Structural Primitives*. Contains canonical interfaces like `Vec2`, `Rect`, `ViewportMatrix`, `WebHTMLNode`, `ResolvedNode`, and state state-machine enums (`InteractionMode`, `ResizeAnchor`, `DragHandleState`).
*   **[matrix.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/matrix.ts)**: *Stateless Math Engine*. Handles zoom anchoring (preventing cursor drift), coordinate projection (`screenToCanvas` and `canvasToScreen`), AABB hit testing, and canvas-space interpolation (`lerp`).
*   **[shadow-mount.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/shadow-mount.ts)**: *DOM Projector*. Owns the `ShadowRoot` lifecycle, reset styles (`SHADOW_RESET_CSS`), absolute/relative wrapper injection (`.canvus-node-wrapper`, `.canvus-flow-child`), and viewport sync via 3D transforms. Triggers reflow boundaries and runs the shared `ResizeObserver` listener. Implements the *Flat String Bridge*.
*   **[tree.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/tree.ts)**: *Hierarchical Tree Model*. Owns `NodeTree`, an in-memory representation of parent-child node relationships. Maintains topological depth indices and coordinates insertion order, sibling structures, ancestor walks, and reparenting safety cycles (preventing circular hierarchies).
*   **[layout.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/layout.ts)**: *Layout Introspection*. Reads `getComputedStyle()` of active Shadow DOM content elements. Detects display types (flex, grid, block) and parses flex orientation axes, wrapping, track heights, grid columns/rows, and layout gaps.
*   **[renderer.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/renderer.ts)**: *Affordance Overlay*. Operates on the HTML5 Canvas 2D Context. Renders primary/secondary element borders, grab handles, margin/padding spacer handles (with coordinate value tooltips), layout mode badges, grid tracks, alignment guides, and marquee selection boundaries.
*   **[drop-zone.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/drop-zone.ts)**: *Flow Placement Estimator*. Detects logical slots inside flex, grid, or block parent containers when dragging a node. Resolves drop target coordinates and draws insertion guide coordinates.
*   **[workspace.ts](file:///Users/balfaro01/Documents/GitHub/canvus/src/workspace.ts)**: *Central Orchestrator*. The public entry point for consumer integrations. Hooks up mouse/pointer event bindings, handles wheel/keyboard shortcuts, maintains active workspace states (panning, resizing, dragging, reparenting, spacing adjustment, and marquee selection), and manages the *Synchronous Reflow Loop*.

## 3. Core Technical Invariants & Interaction Cycles

### A. The Synchronous Reflow Loop
Canvus avoids delayed event busses by running a direct, same-thread layout cycle during user drags or resizes:
```
[Pointer drag movement]
    â”‚
    â–Ľ
1. Style Surgery: Mutate target wrapper `.style.width`/`.style.height` or inline styles.
    â”‚
    â–Ľ
2. Browser Reflow: Browser recalculates block alignments, wrapping, and sibling shifting.
    â”‚
    â–Ľ
3. Observer Read: ResizeObserver fires, measuring the updated bounds of affected nodes.
    â”‚
    â–Ľ
4. Redraw Pass: Bounds cache updates, alignment snapping computations run, overlay canvas redraws.
```

### B. The Flat String Bridge
When a drag or resize gesture completes (`pointerup`):
1. The SDK calls `ShadowMount.extractHTML(id)`.
2. It clone-copies the content node and recursively searches for nested child wrappers (`.canvus-node-wrapper`).
3. It strips away all SDK wrapper chrome, restoring the original DOM element tag structure.
4. The cleaned unpolluted HTML string fragment is returned, which the parent shell commits directly to its AST parser or repository storage.

### C. Drill-Down Selection Flow (Figma-Style Selection)
*   **Single Click:** Selects the topmost selectable item in the currently active nested container scope. Clicking outside exits the current scope and selects the root-level container parent.
*   **Double Click:** Drills down one level into the selected parent's child tree hierarchy.
*   **Cmd/Ctrl Click:** Bypasses tree depth restrictions and directly deep-selects the leaf node under the cursor.
*   **Shift Click:** Add/remove items to a multi-selection set.

### D. Spacing Adjusters (Margins & Paddings)
When a single node is selected, its padding boundaries can be hovered and dragged. Spacing adjusters translate drag movements directly into CSS padding styles, updating child sizes, reflowing contents, and rendering overlay numeric tooltips.

## 4. Developer Guidelines & Constraints

When extending or writing code for Canvus, adhere to these rules:

1.  **Framework Independence**: Do not introduce React, Vue, Svelte, or other UI framework dependencies. Write clean, vanilla, highly optimized TypeScript.
2.  **Coordinate Clarity**:
    *   **Screen space**: Client pixels (`clientX`/`clientY`). Used in raw mouse events and DOM overlay alignments.
    *   **Canvas space (World)**: Scaled and translated viewport units. Used in `NodeTree` caches, bounding boxes (`Rect`), snapping checks, and drawing coordinates.
    *   Always use `screenToCanvas` and `canvasToScreen` from `matrix.ts` for conversions.
3.  **Strict Styling Isolation**: Never apply styles directly inside `ShadowMount` that could bleed or pollute the host. Use `SHADOW_RESET_CSS` inside `shadow-mount.ts` to reset the mount's environment.
4.  **Geometry and Tree Synchronization**:
    *   Never mutate `parentId` or `childIds` of a node directly. Always use the mutation APIs in `NodeTree` (`addNode`, `removeNode`, `reparentNode`, `reorderChild`) to maintain safety invariants.
    *   When mutating DOM structures (e.g., reparenting), make sure to update both `ShadowMount` DOM tree wrappers and the `NodeTree` model.
5.  **Minimize Layout Thrashing**:
    *   Avoid calling `getBoundingClientRect()` repeatedly in hot loops. Use cached values from `currentRect` inside `WebHTMLNode` whenever possible.
    *   Use `suppressObserver` flags in `ShadowMount` during programmatic node manipulations to avoid observer feedback loops.
6.  **Pristine Flat String Exports**: Ensure any new wrapper classes, metadata attributes, or alignment markers are properly cleaned and excluded from `ShadowMount.extractHTML` so the consumer receives completely clean user markup.
